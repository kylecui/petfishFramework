# Inclusion / Exclusion Criteria — LLM Agent SOTA 综述

> 筛选标准与理由记录。确保综述覆盖面与代表性，避免偏差。

## 纳入标准（全部满足）

| # | 标准 | 理由 |
|---|---|---|
| IN1 | **2023-01 至 2026-07** 发表（奠基工作可例外） | agent 领域 ChatGPT 后爆发；旧方法多已被超越 |
| IN2 | **有实证基准结果**（dataset + metric + 对比基线） | 综述目标是找「已验证更优」的方法，非罗列观点 |
| IN3 | **聚焦 LLM-based agent**（非传统 RL/符号 agent） | 与 petfishFramework 技术栈对齐 |
| IN4 | **英文**（arXiv/顶会主流语言） | 覆盖主要学术产出 |
| IN5 | **可复核**（arXiv ID 或 DOI 可查） | 证据可追溯 |

## 排除标准（命中任一即排除）

| # | 标准 | 理由 |
|---|---|---|
| EX1 | 纯观点/立场论文（无实验） | 无法判断方法有效性 |
| EX2 | 仅在作者自建私有数据集验证（无公开基准） | 不可复现、不可比较 |
| EX3 | 已被后续工作明确超越且无独立价值 | 综述关注 SOTA 脉络，非历史全貌 |
| EX4 | 非 agent 主题（纯 NLP 任务如翻译/摘要，无 agent/工具/规划要素） | 超出范围 |
| EX5 | 重复工作（同一团队的扩展版只纳入最新） | 避免冗余 |

## 边界判定（灰区处理）

| 情形 | 判定 | 理由 |
|---|---|---|
| 综述论文（非原创方法） | ✅ 纳入（单独标注为 survey） | 提供分类法与开放问题，是综述的综述 |
| 基准论文（提出新 benchmark） | ✅ 纳入 | 定义 SOTA 标准，是评估基础 |
| 负结果论文（方法失败） | ✅ 纳入 | 失败信号 = gap 信号（skill gotcha 明确要求） |
| 工业 tech report（无同行评审） | ⚠️ 有条件纳入 | 若有强基准则纳入，标注「未经同行评审」 |
| 短文/workshop 未正式发表 | ⚠️ 有条件纳入 | 若被高引或后续正式发表则纳入 |

## 主题覆盖要求（避免偏差）

| 主题 | 最低覆盖 | 代表性要求 |
|---|---|---|
| Agent 架构 | ≥4 个非 ReAct 方法 | LATS、Reflexion、多智能体、自设计各至少 1 篇 |
| RAG | ≥5 个非朴素 RAG 方法 | GraphRAG、Self-RAG、CRAG、Adaptive、Multi-hop 各至少 1 篇 |
| 规划/推理 | ≥4 个非 CoT 方法 | ToT、GoT、LLM+P、PRM/验证 各至少 1 篇 |
| 工具使用 | ≥4 个方法 | ToolFormer/Gorilla、组合、Web Agent、自纠正 各至少 1 篇 |
| 综述/基准 | ≥3 篇综述 + ≥3 个基准 | 覆盖架构分类法 + 主要基准 |

## 生产对照标注规则（RQ6 核心）

每个纳入方法须标注生产状态：

| 标签 | 含义 |
|---|---|
| 🟢 **生产已实现** | LangChain/LlamaIndex/CrewAI 等已内置 |
| 🟡 **部分实现** | 有社区包/实验性支持，但非一等公民 |
| 🔴 **学术独有** | 仅在论文中验证，无生产框架实现（= research-to-engineering gap） |

> 🔴 标记的方法是 petfishFramework 的潜在差异化来源。
