# petfishFramework v1.1.0 开发计划

> 基于 Council-thinking 5+1 分析 + 企业深度评审 P0 反馈
> 设计原则修正：**mode switch（1 个布尔值），不是 8 个 opt-in 旋钮**
> Baseline: v1.0.1, 448 tests, mypy clean, ruff clean

## Council 核心结论

> Critic 的致命攻击：**opt-in P0 security is an oxymoron**。P0 定义为 showstopper，opt-in 意味着可忽略。最危险的是 P0 #5 phantom budget——企业设了 `max_cost_usd` 但 cost 永远 0，guardrail 永不触发，且无错误无警告。

> 综合修正：不是 8 个 opt-in 旋钮，而是 1 个 mode switch。Development mode 保持 5-line quickstart；Enterprise mode（`strict=True`）全部 fail-closed。

## P0 分类与处理

### Type 1: Bug Fix（直接修，0 API 变更）

| ID | 问题 | 修复 | 验收 |
|---|---|---|---|

### Type 2: Mode Switch（`strict=True` 改变全部默认行为）

| ID | strict=False (dev) | strict=True (enterprise) |
|---|---|---|
| #1 Identity | anonymous Subject() (current) | **必填** ExecutionContext，缺失则启动失败 |
| #2 Tool visibility | 全部可见 (current) | **空集默认**，需显式 allowlist |
| #6 Default policy | DefaultAllowPolicy (current) | **拒绝** DefaultAllowPolicy，强制 DenyByDefault 或自定义 |
| #8 Audit redaction | per-sink (current) | **EventEmitter 统一 redaction**，secrets 永不进 event |

**Mode switch API:**
```python
# Development (current 5-liner, unchanged):
agent = Agent(model=..., reasoning=ReAct(), tools=(...))
# Console: WARNING — development mode, not for production use

# Enterprise:
agent = Agent(
    model=..., reasoning=ReAct(), tools=(...),
    strict=True,
    execution_context=ExecutionContext(subject_id="user1", roles=["engineer"]),
)
```

**FIX (Momus blocker #1):** strict 校验在 `Agent.__post_init__` 中执行（非 RuntimeEnvironment），确保 `Agent(strict=True)` 构造时即失败：

```python
def __post_init__(self):
    # ... existing model resolution ...
    if self.strict:
        if self.execution_context is None or self.execution_context.subject_id == "anonymous":
            raise ValueError("strict mode requires non-anonymous ExecutionContext")
        if isinstance(self.permission_policy, DefaultAllowPolicy):
            raise ValueError("strict mode rejects DefaultAllowPolicy")
        if self._budget and self._budget.max_cost_usd and not self._pricing_available:
            raise ValueError("strict mode: max_cost_usd set but pricing unavailable for model")
```

这样 `Agent(strict=True)` 在构造时即失败（不需要等到 `session()`/`run()`）。

### Bug Fix #5: Cost Pricing

创建 `src/petfishframework/models/pricing.py`:

```python
PRICING_TABLE: dict[str, ModelPricing] = {
    "gpt-4o": ModelPricing(input_per_1m=2.50, output_per_1m=10.00),
    "gpt-4o-mini": ModelPricing(input_per_1m=0.15, output_per_1m=0.60),
    "claude-3.5-sonnet": ModelPricing(input_per_1m=3.00, output_per_1m=15.00),
    # ... more models
}

def compute_cost_usd(model_name: str, usage: Usage) -> float | None:
    """Returns USD cost or None if model unknown."""
    pricing = PRICING_TABLE.get(model_name)
    if pricing is None:
        return None
    return (usage.input_tokens / 1_000_000 * pricing.input_per_1m
          + usage.output_tokens / 1_000_000 * pricing.output_per_1m)
```

模型适配器修改：
- `OpenAIModel.query()`: `cost_usd = compute_cost_usd(self.model_name, usage)`
- `Budget`: 如果 `max_cost_usd` 被设置但 `cost_usd is None` → strict 模式启动失败；dev 模式 warn

修复 `examples/05_enterprise_expense.py`：

### Mode Switch: ExecutionContext

创建 `src/petfishframework/core/context.py`:

```python
@dataclass(frozen=True)
class ExecutionContext:
    """Immutable identity context for a session.
    
    In strict mode, Agent rejects construction without this.
    In dev mode, defaults to anonymous.
    """
    subject_id: str = "anonymous"
    roles: tuple[str, ...] = ()
    tenant_id: str | None = None
    trace_id: str | None = None
    
    @staticmethod
    def anonymous() -> ExecutionContext:
        return ExecutionContext()
```

Agent 新增字段：`execution_context: ExecutionContext | None = None`。
Session 传递到 RuntimeEnvironment。
`_prepare_tool_call` 使用 `self.execution_context.subject` 而非空 `Subject()`。

### Mode Switch: Tool Visibility (strict)

RuntimeEnvironment 新增 `tool_filter: Callable[[list[Tool]], list[Tool]] | None = None`。

- `strict=False`: `tools()` 返回全部（current）
- `strict=True` + `tool_filter=None`: 启动失败（必须配置 filter）
- `strict=True` + `tool_filter=set_filter({"calculator", "search"})`: 只返回 allowlist 中的工具

简化 API：`tool_filter` 可以是 `set[str]`（tool names）或 `Callable`。

### Mode Switch: Strict Validation（仅在 Agent.__post_init__）

strict 校验**只在** `Agent.__post_init__` 中执行（见上方 Fix #1 代码）。RuntimeEnvironment 不重复校验——Agent 构造时已保证不变量，Session/RuntimeEnvironment 继承已验证的 Agent 状态。

### Mode Switch: Unified Audit Redaction (strict)

EventEmitter 新增 `redact_fn: Callable[[Event], Event] | None = None`。

- `strict=False`: `redact_fn=None`（current behavior，per-sink redaction 如 SIEMSink）
- `strict=True`: 默认 redact_fn = `_default_redactor`，移除 `_credential_token`、`api_key`、`password`、`secret`、`token` 字段值

```python
def emit(self, type, data):
    event = Event(type=type, timestamp=time.time(), data=data)
    if self._redact_fn:
        event = self._redact_fn(event)
    # ... rest unchanged
```

### Feature #3: ApprovalStateMachine (minimal)

创建 `src/petfishframework/permissions/approval.py`:

```python
class ApprovalStatus(Enum):
    PENDING = "pending"
    APPROVED = "approved"
    DENIED = "denied"
    EXPIRED = "expired"
    CONSUMED = "consumed"

@dataclass
class ApprovalRequest:
    request_id: str
    session_id: str
    tool_name: str
    args_hash: str  # SHA256 of args
    policy_version: str
    status: ApprovalStatus = ApprovalStatus.PENDING
    created_at: float = field(default_factory=time.time)
    expires_at: float = 0.0  # 0 = no expiry
    approver: str | None = None

class InMemoryApprovalStore:
    """Agent-scoped approval store, shared across all sessions of an Agent."""
    def create(self, session_id, tool_name, args_hash, policy_version) -> ApprovalRequest: ...
    def approve(self, request_id, approver: str) -> ApprovalRequest: ...
    def deny(self, request_id, reason: str) -> ApprovalRequest: ...
    def consume(self, request_id) -> bool: ...  # one-time, returns False if already consumed
    def get(self, request_id) -> ApprovalRequest | None: ...
```

RuntimeEnvironment 行为变更：
- REQUIRE_APPROVAL + `approval_store=None` → DENY（不再返回 error，fail-closed）
- REQUIRE_APPROVAL + `approval_store=set` → 创建 ApprovalRequest(PENDING)，返回 ToolResult(error="approval_required: {request_id}")

**FIX (Momus blocker #3):** ApprovalStore 是 **Agent 级别**（非 session-scoped），通过 Agent → Session → RuntimeEnvironment 传递：

```python
# Agent 持有 approval_store（共享于该 Agent 的所有 Session）
agent = Agent(model=..., approval_store=InMemoryApprovalStore())

# Agent.approve/deny 操作 Agent 级 store
agent.approve(request_id, approver="manager1")  # → store.approve(...)
agent.deny(request_id, reason="too risky")      # → store.deny(...)

# Session 从 Agent 继承 approval_store，传给 RuntimeEnvironment
session = agent.session(task)
# session._env.approval_store == agent.approval_store
```

这样 `agent.approve()` 能直接定位到 store，无需 session 引用。
