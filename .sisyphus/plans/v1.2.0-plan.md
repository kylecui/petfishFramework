# petfishFramework v1.2.0 开发计划

> Baseline: v1.1.0, 484 tests, mypy clean, ruff clean
> 原则：progressive disclosure — 所有新 feature 是 opt-in/additive；默认行为与 v1.1.0 字节一致
> Scope: P1-01…P1-07, P1-09, P1-10 + P2-01, P2-02, P2-04, P2-05, P2-06（P1-08, P2-03 标注, P2-07 已完成）

## 设计决策

### D1 — CompiledContext (P1-01): 可插拔 ContextCompiler 协议
独立 protocol（非 Session 方法）。`Session._prepare_run` 当前硬编码空 CompiledContext（session.py:73-78）。修改：Agent 新增 `context_compiler: ContextCompiler | None = None`；Session 在设置时调用 compiler(task, execution_context, memory, retriever)，否则回退到当前空默认。ExecutionContext 是编译输入。默认行为不变。

### D2 — 持久化回放 (P1-03): 可插拔 EventStore + JSONL 默认
`EventStore` protocol (append/get/since/checkpoint)。`JsonEventStore` (JSONL 文件, stdlib only) 为默认持久化实现；`InMemoryEventStore` 保持当前行为。`EventEmitter(store=...)` 可选参数（默认 = in-memory = 无行为变更）。不使用 SQLite（重、二进制、非 git-friendly）。

### D3 — MCP HTTP (P1-04): MCPTransport protocol + httpx extra
框架当前自己实现 JSON-RPC（非官方 MCP SDK）。保持一致：提取 `MCPTransport` protocol；StdioMCPClient 适配；新增 `StreamableHttpMCPClient` 基于 optional extra `mcp-http`（httpx）。提供 `connect_http()` factory。

### D4 — 容器沙箱 (P1-06): 可插拔 SandboxBackend + docker extra
`SandboxBackend` protocol。当前 subprocess 逻辑变为 `SubprocessSandboxBackend`（默认）；`DockerSandboxBackend` 基于 optional extra `sandbox-docker`。SandboxExecutor 变为 facade。默认路径不变。

### D5 — RAG 授权 (P1-07): 独立 RetrievalPolicy，与 PermissionPolicy 可组合
`ResourceMetadata` dataclass（clearance, tenant_id, allowed_roles）附加到 retriever 输出。`RetrievalPolicy` protocol 在 `Environment.retrieve()` 中检索后、返回前执行过滤。默认 `AllowAllRetrievalPolicy` = 当前行为。PermissionPolicy 保持工具调用聚焦，不合并。

### D6 — CapabilityCatalog (P1-10): 共存，不替换
`CapabilityCatalog` 统一 native tools（explicit + ToolRegistry）和 MCP tools。Agent 新增 `capabilities: CapabilityCatalog | None = None`。设置时替代 tools/tool_registry 解析；None 时当前路径不变。Catalog 暴露 `resolve(task) -> tuple[Tool, ...]` + `all_tools()`。

### D7 — 错误码 (P1-09): ToolErrorCode enum on subclasses
`ToolErrorCode(str, Enum)` 作为每个 ToolExecutionError 子类的 class attribute。ToolResult 新增 `error_code: str | None`。现有字符串消息不变（additive）。

### D8 — FastAPI/CLI (P2-01): 独立 server extra
新增 `src/petfishframework/server/` 包。`create_app(agent) -> FastAPI`（/run, /session, /health）。`petfishframework serve` CLI 子命令。fastapi + uvicorn 基于 optional extra `server`。

## Wave 分解

### Wave 1 — 快速胜利（独立，低风险）
- **P1-09 错误码** → `core/errors.py` (+ToolErrorCode), `core/types.py` (ToolResult.error_code), env maps codes
- **P1-02 AgentAsTool identity** → `tools/agent_tool.py` 接受/传递 ExecutionContext

### Wave 2 — 基础协议（独立）
- **P1-01 CompiledContext** → `core/compiler.py` (ContextCompiler protocol + DefaultContextCompiler)
- **P1-03 EventStore** → `core/event_store.py` (EventStore protocol, InMemoryEventStore, JsonEventStore)
- **P1-05 SecretProvider** → `credentials/provider.py` (SecretProvider protocol); CredentialBroker refactored behind it

### Wave 3 — 环境集成（触及咽喉点）
- **P1-07 RAG 授权** → `retrieval/policy.py` (ResourceMetadata, RetrievalPolicy, AllowAllRetrievalPolicy)
- **P1-10 CapabilityCatalog** → `tools/catalog.py` (CapabilityCatalog); Agent.capabilities

### Wave 4 — 可选依赖功能（隔离在 extras 后）
- **P1-04 MCP HTTP** → `mcp/transport.py` (MCPTransport protocol); `mcp/http_transport.py` (StreamableHttpMCPClient, extra mcp-http)
- **P1-06 容器沙箱** → `tools/sandbox_backend.py` (SandboxBackend protocol, SubprocessSandboxBackend); `tools/docker_sandbox.py` (DockerSandboxBackend, extra sandbox-docker)

### Wave 5 — P2 基础设施/工具
- **P2-01 FastAPI/CLI** → `server/` 包 + `__main__.py serve` (extra server)
- **P2-02 兼容矩阵** → `docs/compatibility-matrix.md`
- **P2-04 策略 lint/simulate** → `scripts/policy_lint.py` + `scripts/policy_simulate.py`
- **P2-05 SBOM/签名/扫描** → `.github/workflows/supply-chain.yml` (cyclonedx-py, sigstore, pip-audit)
- **P2-06 性能基准** → `benchmarks/` 目录 + `docs/benchmark-results.md`

### Wave 6 — 发布
版本号更新（pyproject, __init__, api.md, CHANGELOG, README badge, **SECURITY.md 1.2.x 行**），api-stability.md tier 更新，pre_release.py 1.2.0。

## 依赖分析

```
Wave 1 (P1-09, P1-02): 互相独立，与一切独立
Wave 2 (P1-01, P1-03, P1-05): 各自独立（P1-01 依赖 ExecutionContext v1.1 已有）
Wave 3 (P1-07, P1-10): 各自独立，触及不同 chokepoint
Wave 4 (P1-04, P1-06): 自包含在 mcp/ 和 tools/
Wave 5 (P2-*): 全部独立基础设施，P1 稳定后执行
Wave 6: 严格最后
```
无跨 Wave 硬序约束（全部设计为 additive）。
