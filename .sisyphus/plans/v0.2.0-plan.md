# petfishFramework v0.2.0 开发计划

> 基于 5 轮用户反馈 + 生产可用路线图 + v0.1.9 状态
> 目标：从 Alpha runtime control skeleton 进入企业 PoC 可用阶段
> 原则：**不堆 feature，聚焦企业 PoC 可信度闭环**

## v0.2.0 使命

> 让一个企业开发者能在 30 分钟内：安装 → 配置权限策略 → 运行一个受控 Agent → 导出审计报告。

## 任务清单

### Phase A: 企业 PoC Demo（证明差异化，非 calculator）

#### A1: Enterprise Expense Approval Agent（端到端 demo）

**目标**：一个完整可运行示例，展示全部 6 种 DecisionEffect + Budget + AuditReport。

**文件**：`examples/05_enterprise_expense.py`

**场景**：报销审批 Agent，处理员工报销请求。

**覆盖**：
- 报销金额 < $500 → ALLOW（auto-approve）
- 报销金额 $500-$5000 → REQUIRE_APPROVAL
- 报销金额 > $5000 → DEGRADE 到 dry-run（无 fallback → fail-closed）
- 员工 SSN → MASK（input + output + event）
- 非财务角色 → DENY 特定工具
- Budget 限制总 token 消耗
- AuditReport 导出 Markdown + JSON

**TDD 测试先行**：

```python
# tests/test_enterprise_demo.py

def test_expense_under_500_auto_approved():
    """$350 team lunch → ALLOW, auto-approved."""

def test_expense_500_to_5000_requires_approval():
    """$2000 conference → REQUIRE_APPROVAL."""

def test_expense_over_5000_degrade_fail_closed():
    """$8000 equipment → DEGRADE, no fallback → blocked."""

def test_ssn_masked_in_input_output_and_event():
    """SSN masked before tool, in result, and in audit log."""

def test_non_finance_role_denied():
    """Non-finance role cannot call approve_payment."""

def test_budget_enforced():
    """Total token budget enforced."""

def test_audit_report_generated():
    """Markdown audit report contains all decisions."""
```

**实现**：
- PolicyChecker tool（金额检查）
- ApprovePayment tool（审批工具，side_effect=True）
- ExpensePolicy（使用 tool metadata 做决策）
- FakeModel 驱动（无需 API key）

#### A2: Structured Audit Report 增强

**目标**：AuditReport 从 MVP 升级为可读、可审计的完整报告。

**新增字段**：
- Budget section（used vs limit）
- Event count by type
- Permission count by effect
- Masked fields summary
- Degraded calls summary
- Blocked calls summary
- Model call count + token summary
- Error section
